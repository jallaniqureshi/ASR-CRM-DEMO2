Publishing the ASR CRM demo

Three ways to put this prototype online so colleagues, brokers or stakeholders can open a link and have a play — plus what to do before it becomes a real production system.

The CRM is a self-contained web app — a set of HTML, CSS and JavaScript files with no server or database required. That makes it trivial to host on any static-site service for free. Pick the route that suits you.

Before you share: the app saves each visitor's edits in their own browser, so everyone gets a clean, private copy of the sample data. There's a Reset demo data button inside the app (open the About this demo panel via the ⓘ icon).

1Netlify Drop

The fastest path — a live URL in under a minute, no account setup, no command line.

Drag-and-drop hosting Easiest · recommended

  1. Download this project as a folder (use the download option in the chat, or export the whole project).
  2. Go to app.netlify.com/drop.
  3. Drag the entire project folder onto the page.
  4. Netlify gives you an instant URL like asr-crm-demo.netlify.app — share it with anyone.
  5. To update later, drag the folder again, or connect it to a GitHub repo for automatic deploys.

Make sure the folder you drop contains ASR CRM.html at its top level. Rename that file to index.html first if you want the URL to open the app directly without the filename on the end.

2Vercel or GitHub Pages

A more permanent home with a memorable address and easy updates — ideal once the demo is being shared more widely.

Vercel Free

  1. Create a free account at vercel.com.
  2. Click Add New → Project and import (or drag) the project folder.
  3. No build settings needed — it's a static site. Click Deploy.
  4. You get a .vercel.app URL and can add a custom domain.

GitHub Pages Free

  1. Create a GitHub repository and upload the project files.
  2. Rename ASR CRM.html to index.html.
  3. In Settings → Pages, set the source to your main branch.
  4. Your site appears at yourname.github.io/repo.

3The single-file version

Everything — code, styles, fonts, logos — packed into one HTML file.

ASR CRM (standalone).html Offline-capable

This file works on its own with no other files alongside it. Use it to:

  • Email the demo to someone — they double-click to open it, even offline.
  • Drop it onto Netlify/Vercel on its own for a one-file site.
  • Put it on a shared drive or SharePoint.

Trade-off: it's a single large file, so updates mean re-exporting it. For an evolving demo, host the folder (options 1–2) instead.

Making it production-ready

The demo is built to be readable and editable. A few steps turn it into something robust enough for ongoing internal use.

  • Pre-compile the code. The demo compiles its JavaScript in the browser on each load (fine for a demo, slightly slower to start). For production, pre-build it with a tool like Vite so it loads instantly. Your developer / IT team can do this from the same source files.
  • Add a login. Netlify, Vercel and Cloudflare all offer password protection or single-sign-on so only invited people can open the link — useful while it contains placeholder data.
  • Connect real data. Today the app uses illustrative sample data held in the browser. A production version would read and write to a database or to your existing systems (e.g. Navigator PAS, Power BI) through an API.
  • Use a custom domain. All three hosts let you point an address like crm-demo.africaspecialtyrisks.com at the site.
